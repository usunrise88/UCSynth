# GUI-контроллер (этап 2)

Десктоп-пульт `app/` — управляет синтом по тому же бинарному протоколу, что и `serialtest.py`, но
мышью и клавиатурой. **Стек:** чистый Go + [Gio](https://gioui.org) (без cgo на Windows). **Таргет:**
Windows; собирается и на Linux (кросс-компиляция).

## Что в v1 (ядро-контроллер, VST-облик)
- **Подключение** по COM-порту; авто-подсветка нативного USB-порта S3 (Espressif VID 303A) — не мост CH343.
- **Панельная сетка a-la VST** (синий акцент, аутлайн-контролы): 3 колонки панелей из блоков реестра.
  Контролы по типу — **радиальные ручки** (float), **вертикальные слайдеры** ADSR, **степперы** (int),
  **сегменты** (enum), **тумблеры** (bool). Панели строятся из дампа `LIST`; двусторонняя синхра (крутишь —
  слышно, клампы с устройства отражаются), новый параметр появляется сам (неизвестные — в панель «Прочее»).
- **Экранная клавиатура:** мышь + ПК-клавиши по раскладке **FL Studio / musical typing** (нижний ряд
  `Z X C V B N M , . /` = белые, `S D · G H J · L ;` = чёрные; верхний `Q…P` на октаву выше), октава `↑`/`↓`
  или кнопки «◂ Окт / Окт ▸», кнопка **Panic** в верхнем баре.
- **Метрики** в реальном времени: CPU / underruns / heap / uptime (из `STAT`, ~2 Гц). Клик по метрикам —
  **графики истории** (спарклайны CPU / heap / underruns, кольцо ~1 мин).
- **Баннер тест-тона:** `test_tone` включён по умолчанию и перебивает ноты — баннер это показывает и даёт выключить.

## Вкладки (слои поверх ядра)
- **Синт** — панельная сетка (выше) + клавиатура.
- **Секвенсор** — piano-roll: сетка шаги×высоты (16 шагов, 2 октавы), клик ставит ноту, транспорт (играть/стоп/
  очистить), темп; плеер тактируется в отдельной горутине и шлёт NOTE_ON/OFF (ноты идут через race-safe `noteSink`).
- **Патчи** — JSON-файлы на ПК (дерево папок под `%AppData%/UCSynth/patches`): сохранить (снимок всех значений),
  загрузить (SET всех, неизвестные имена пропускаются), удалить, импорт/экспорт по пути к файлу.

**MIDI-вход** (нижний бар): выбор устройства → ноты идут в синт. Транспорт — **WinMM через stdlib `syscall`**
(cgo-free, только Windows; на других ОС — заглушка). Приложение собирается как **GUI** (`-H=windowsgui`) — без консоли.

## Собрать
Go ≥ 1.25 (`app/go.mod`). Из корня репозитория:

```bash
make app-build     # → app/build/ucsynth-controller.exe (Windows, кросс из Linux)
```

Нативно на Windows:

```powershell
cd app
go build -o ucsynth-controller.exe ./cmd/controller
```

## Запустить (Windows, с железом)
1. Прошей плату (см. `docs/build-flash.md`).
2. Запусти `ucsynth-controller.exe`.
3. Нажми **Порты**, выбери порт S3 (подсвечен «UCSynth / Espressif»), **Подключить**.
4. Блоки заполнятся из реестра. Баннер попросит выключить тест-тон — выключи, играй с клавиатуры.

**Грабли:**
- Портов может быть два (нативный USB-JTAG и мост CH343). Протокол — на **нативном** (Espressif VID).
- Открытие порта держит DTR/RTS low, чтобы **не ресетить** S3 (как `serialtest.py`). Если плата всё же
  ресетится на подключении — сообщи.
- Тест-тон по умолчанию ВКЛ (перебивает ноты) — выключи через баннер.

## Разработка / проверка (без железа и дисплея)
```bash
make app-test      # чистые пакеты (go test) + кросс-vet/сборка Windows (cgo-free)
```
Gio-UI в облаке не запускается (headless, нужен Windows-бинарь) — проверяется **кросс-сборкой**;
логика (протокол/модель) — нативными тестами (в т.ч. `device` против fake-прошивки через `net.Pipe`).

## Структура `app/`
| Пакет | Роль | Тест |
|-------|------|------|
| `proto/` | кадры, CRC-16/CCITT-FALSE, кодек сообщений | golden-векторы (0x29B1), resync |
| `serial/` | перечень портов, open, DTR/RTS (go.bug.st/serial) | — (I/O) |
| `device/` | клиентский реестр из LIST, приоритетный writer (ноты > SET > STAT), throttle, echo | fake-прошивка (net.Pipe) |
| `layout/` | карта параметр→блок + подписи enum/единицы (прошивка их не шлёт) | да |
| `patch/` | патчи JSON (name→value), Store = дерево .json-файлов, save/load/import/export | да (tmpdir, обход дерева, traversal) |
| `seq/` | step-секвенсор: сетка + плеер с клоком (шаг → NOTE_ON/OFF) | да (advance/wrap/toggle, виртуальный клок) |
| `midi/` | парсер MIDI-сообщений (чистый) + WinMM-транспорт (`//go:build windows`, cgo-free) + заглушка | парсер — да; транспорт — только сборка |
| `ui/` | Gio: кастомные виджеты (ручка/слайдер/pill), панельная сетка, вкладки, patches/seq/midi/графики, клавиатура, тема | headless-тесты (роутинг, драг ручки, клик по сетке, smoke всех вкладок) + кросс-сборка |
| `cmd/controller/` | окно + связка | кросс-сборка Windows |

Контракт протокола — `docs/serial-protocol.md`. Референс-клиент на Python — `tools/serialtest.py`.
