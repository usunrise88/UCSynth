# Гайд: OLED SSD1306 (GME18864) по I2C — отладочный экран

Монохром 128×64 на SSD1306, I2C-адрес **0x3C**. Временный отладочный экран (splash, осциллограф,
значения параметров) до цветного ST7796 + тач (этап 9). I2C-шина **общая** — позже сюда VL53L0X
(0x29) и MCP-расширители.

## 1. Подключение — пин-в-пин

| OLED (GME18864) | ESP32-S3 | Примечание |
|-----------------|----------|-----------|
| VCC | 3.3V | модуль 3.3В (не 5В!) |
| GND | GND | общая земля |
| SDA | GPIO 8 | I2C данные |
| SCL | GPIO 9 | I2C клок |

Внутренние подтяжки S3 включены в коде; на большинстве GME-модулей есть свои SMD-подтяжки на
SDA/SCL — на коротких проводах хватает. Если шина не видит экран или сбоит — внешние 4.7 кОм
SDA→3.3V и SCL→3.3V. Проверь по `progress.md`: GPIO 8/9 не заняты (I2S — 5/6/7).

## 2. Грабли этого модуля

> - **Адрес 0x3C** (не 0x3D). Если на плате перемычка выбора адреса — проверь её.
> - **Питание 3.3В** — 5В на VCC не подавать.
> - Если контроллер окажется **SH1106** (частый клон под 1.3"): картинка сдвинута на ~2 px и мусор
>   по краю. Лечится сдвигом колонок в драйвере (скажи — поправлю).

## 3. Проверка до кода

- Мультиметр: на VCC 3.3В, GND общий с S3.
- После прошивки в boot-логе — `display: OLED SSD1306 128x64 @ 0x3C ...`. Если вместо этого
  `OLED не найден на 0x3C` — I2C не ответил (см. диагностику), но звук при этом работает.

## 4. Диагностика — «симптом → проверить»

**Экран тёмный / пустой:**
1. Boot-лог `SSD1306 128x64 init OK`? Нет → нет ACK по I2C: SDA=8/SCL=9 не перепутаны? питание 3.3В? адрес 0x3C?
2. init OK, но пусто → редкость при нашей init-послед.; убедись, что это SSD1306.
3. SDA/SCL местами — типовая ошибка, поменяй.

**Картинка сдвинута / мусор по краю:** это SH1106 → нужен сдвиг колонок (скажи).

**Мерцание / артефакты:** длинные провода I2C на бредборде — укороти или добавь внешние подтяжки 4.7 кОм.
