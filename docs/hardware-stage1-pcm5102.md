# Гайд: PCM5102 по I2S (этап 1.1)

Модуль **GY-PCM5102** (I2S DAC, minijack распаян) → ESP32-S3. Один модуль из пяти.
Формат тракта: I2S standard (Philips), 48 кГц, 16 бит, стерео-слот (mono дублируется в L+R).
Инвентарь и общие заметки по железу — `docs/hardware.md`.

## 1. Подключение — пин-в-пин

| PCM5102 | ESP32-S3 | Примечание |
|---------|----------|-----------|
| BCK  | GPIO 5  | I2S bit clock |
| LCK  | GPIO 6  | I2S word select (LRCK) |
| DIN  | GPIO 7  | I2S data (S3 → модуль) |
| SCK  | **GND** | внутренний PLL из BCK; висящий SCK = тишина |
| XSMT | **3.3V** | снять soft-mute (на GND = вечная тишина) |
| FMT  | **GND** | стандартный I2S (не left-justified) |
| FLT  | —       | по умолчанию, не трогать |
| DEMP | —       | по умолчанию, не трогать |
| VIN  | 3.3V    | отдельный LDO под аудио (AMS1117); развязка 100нФ + 10–47мкФ у VIN |
| GND  | GND     | star ground: аналог и цифра сходятся в одну точку, не в петлю |

MCLK со стороны S3 **не заводится** — в конфиге I2S `gpio_cfg.mclk = I2S_GPIO_UNUSED`.
Пины 5/6/7 выбраны вне strapping (0/3/45/46), USB-JTAG (19/20) и зоны Octal-PSRAM (33–37);
занесены в карту пинов `progress.md`.

> **GY-PCM5102A:** XSMT/FMT/FLT/DEMP выведены не на гребёнку, а на **тыльные площадки**. На
> типовой плате дефолты уже верные — XSMT подтянут на 3.3В (не в мьюте), FMT на GND (I2S);
> проверено прозвонкой. Паять эти пады не нужно, если прозвонка совпадает с таблицей.

## 2. Грабли PCM5102

> - **SCK → GND** — причина №1 «нет звука». Не на S3, а на землю.
> - **XSMT → 3.3V** — иначе soft-mute: клоки идут, а на выходе тишина.
> - **FMT → GND** — стандартный I2S; иначе сдвиг битов / мусор.
> - Аналоговое питание PCM5102 — **отдельным LDO** от цифры (MB102 импульсный, шумит).

## 3. Проверка до кода

Мультиметром/анализатором **до** запуска прошивки:
- Мультиметр: XSMT ≈ 3.3 В при поданном питании (не 0).
- Мультиметр: SCK звонится на GND; FMT на GND.
- Мультиметр: VIN ≈ 3.3 В.
- Анализатор на BCK/LCK/DIN: после старта прошивки BCK ≈ 1.54 МГц (48к×16бит×2 слота),
  LCK ≈ 48 кГц, на DIN — данные (не константа).

## 4. Диагностика — «симптом → проверить»

**Тишина:**
1. SCK на GND? ← причина №1.
2. XSMT ≈ 3.3 В (не в мьюте)?
3. Анализатор: идут ли BCK/LCK?
   - нет → проблема в конфиге I2S (пины, `i2s_channel_enable`, не тот порт);
   - да, но DIN пустой → аудио-задача не пишет (DMA/задача на Core 0 не стартовала).
4. VIN есть? Земля общая с S3 (star ground)?
5. Minijack воткнут до конца; в логе `audio: I2S TX ... синус (этап 1.1)`?

**Хрип / прерывистый звук:**
1. Underruns — аудио-задача не успевает (метрики появятся в 1.2).
2. Просажено/шумит аналоговое питание; общая земля петлёй.
3. Плохой контакт BCK/DIN на бредборде (длинные провода на 1.5 МГц).

**Писк не по частоте / искажение:**
1. FMT не на GND — формат сдвинут.
2. Перепутаны BCK и LCK.
