# Спецификация

Полифонический wavetable-синтезатор на ESP32-S3.

**Статус:** проба пера. Разовая обучающая сборка из готовых модулей. Не фундамент линейки — настоящая платформа будет на STM32 отдельным проектом.

Железо: см. `hardware.md`. План работ: см. `roadmap.md`.

---

## 1. Звуковой движок

### Голос
| Блок | Спецификация |
|------|--------------|
| Осцилляторы | 3 на голос, wavetable |
| Микшер | Уровни 3 осц + шум + ring mod (осц1×осц2) |
| Фильтр | State-variable (LP/HP/BP), cutoff + resonance |
| Огибающие | ADSR (VCA), ADSR (VCF) |
| Wave-огибающая | 8-точечная, зацикливаемая → позиция в wavetable |

### Глобальное
| Блок | Спецификация |
|------|--------------|
| Полифония | Цель 8 голосов (старт 1, масштабируем) |
| LFO | 2×, свободная маршрутизация |
| Эффекты | Overdrive → Delay → Reverb |
| Матрица модуляции | Источники → приёмники, гибкая |

### Режимы осциллятора
- **Clean** — band-limited wavetable, антиалиасинг
- **Lo-fi** — bit-reduction до 8 бит, без антиалиасинга (алиасинг как фича)

### Матрица модуляции
**Источники:** LFO1, LFO2, ADSR-VCF, Wave-env, VL53L0X, velocity, mod-wheel
**Приёмники:** pitch, wavetable position, cutoff, resonance, amp, FX-параметры

### Формат
- float DSP, 44.1 / 48 кГц
- Блочная обработка (32–128 семплов)

## 2. Управление

### Физические контролы
| Контрол | Кол-во | Назначение |
|---------|--------|------------|
| Матрица 4×4 | 16 | Ноты / шаги секвенсора |
| Энкодеры осциллятора | 4 | 2 с кнопкой (выбор осц / тип волны) |
| Utility-энкодеры | 4 | ADSR / LFO / FILTER / FX, с кнопками |
| Энкодер управления | 1 | Master volume + навигация, с кнопкой |
| Кнопки | 3 | Play/Pause, Menu, Shift |
| Тачскрин | 1 | Навигация, визуализация, XY-пад |
| ToF-сенсор | 1–2 | Бесконтактная модуляция |

**Итого:** 9 энкодеров (7 с кнопкой), 19 кнопок, тач, ToF.

### Режимная модель
- **note-mode** — матрица играет ноты, scale-mode (по гаммам), октавный сдвиг
- **seq-mode** — матрица = 16 шагов, подсветка на экране
- **p-lock** — зажать шаг + крутить энкодер → значение параметра только для этого шага
- **Shift-слой** — второй смысл каждой кнопки/энкодера

**Velocity** — параметр шага (p-lock), а не сила нажатия (кнопки её не дают).

### Разделение тач / энкодеры
- **Энкодеры** — точные значения параметров (крутишь вслепую)
- **Тач** — навигация, пресеты, визуализация, XY-жесты, рисование огибающих
- Не дублировать: экран показывает, что делают энкодеры

## 3. Управление с ПК (GUI)

**Стек:** Go, Windows. **Протокол:** USB CDC (Serial).

### Роль
1. **В разработке** — единственный пульт, пока нет физических контролов. Отладка DSP без единого энкодера.
2. **После** — редактор пресетов, визуализация, отладочный монитор.

### Требования
- Чтение/запись любого параметра
- Триггер нот (note-on/off) с velocity
- Загрузка/выгрузка пресетов
- Загрузка пользовательских wavetable
- Мониторинг: CPU, underruns, свободная память

### Протокол
Текстовый на старте (проще отлаживать).

Команды: `GET <id>`, `SET <id> <val>`, `LIST` (дамп реестра), `NOTE_ON <n> <vel>`, `NOTE_OFF <n>`, `STAT`.

**`LIST` — ключевая команда:** GUI строит интерфейс **динамически** из дампа реестра. Новый параметр в прошивке не требует правки Go-кода.

## 4. Вне скоупа

- Много движков синтеза (ELZ_1 style) — только wavetable + lo-fi режим
- Гранулярный синтез, convolution reverb — CPU не тянет
- Аналоговый фильтр (SSI 2144) — путь будущей STM32-платформы
- Velocity-клавиатура (keybed) — динамика через p-lock
- Своя печатная плата — готовые модули + перфоборд
- MIDI DIN — только USB-MIDI и кнопки
