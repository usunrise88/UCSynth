Изначально пин был v5.5.4; подняли до v6.0.2, чтобы совпасть с локальной установкой пользователя (на ней и прошли верификацию).

- Тулинг «минимум ручных команд»: `tools/setup-esp-idf.sh` (version-aware), `tools/build.sh`, `Makefile` (`make build/menuconfig/size/merge/flash`).

Проверено **на железе** (ESP32-S3-N16R8, ESP-IDF v6.0.2): boot-лог показал `flash 16 МБ`, `PSRAM 8 МБ` (Octal, `SPI SRAM memory test OK`), все слои `*_init`, `boot complete` + heartbeat `alive`. **Этап 0.1 закрыт ✅.**

Наблюдение по прошивке (не блокер): на плате пользователя два USB — нативный USB-Serial-JTAG (сюда идут наши логи) и мост CH343. Авто-сброс после `flash` не всегда возвращает чип в run — иногда нужен ручной **RST**. Занесено в `docs/build-flash.md`.

### 2026-07-15 — этапы 0.2 + 0.3: модель параметров + бинарный протокол
**0.2 — реестр (`control`):** статический массив по `enum ParamId`, метаданные const (во flash), значения `std::atomic<float>` — lock-free (Core 1 пишет, Core 0 читает без мьютекса). API `set_param`/`get_param`/`param_count`/`param_get_info`. Сид: `master_volume`, `test_tone_hz`.

**0.3 — бинарный протокол (`comm`):** формат выбран пользователем (эффективность важнее читаемости). Кадр `[55 AA][LEN][BODY][CRC16]`, CRC-16/CCITT-FALSE — демукс из ASCII-логов на общем USB-JTAG-канале. Опкоды `GET/SET/LIST/NOTE_ON/NOTE_OFF/STAT`. Транспорт: `usb_serial_jtag` драйвер + задача на **Core 1**, сырое чтение/запись (без stdio/float-printf). Логика (`frame.*`/`protocol.*`) отделена от I/O → host-тестируема. NOTE_ON/OFF пока только ACK+лог (голоса — этап 3).

Проверено: **сборка в облаке** (`ucsynth.bin` ~196 КБ); **host-тест** `tools/run-host-tests.sh` (реестр, кадрирование, CRC, диспетчер; CRC check value `0x29B1` = паритет прошивки с GUI/скриптом). Долг: **D-006** (бинарь), **T-003** (общий канал log+протокол).

Контракт — `docs/serial-protocol.md` (по нему пишется Go-GUI, этап 2).

**Проверено на железе** (2026-07-15, нативный USB COM9): `python tools/serialtest.py` прошёл — LIST→2 параметра, GET/SET с эхо-подтверждением, SET 9→кламп 1.0, STAT (heap ~8.3 МБ, `heap == minheap` → память с загрузки не колыхалась, uptime растёт), NOTE_ON→ACK. Все ответы совпали с `docs/serial-protocol.md`. Мусора в потоке нет — демукс лог+протокол по CRC отработал. **Этап 0 закрыт целиком ✅**, дальше этап 1 (звук: I2S + PCM5102 + wavetable-осц).

### 2026-07-15 — этап 1.1: I2S + PCM5102 ✅ на железе
Аудио-тракт: I2S **standard** TX (Philips, 48 кГц, 16 бит, стерео-слот) на **Core 0**, вывод на PCM5102. Аудио-задача (prio 10, Core 0) генерит блок 64 семпла и блокируется на `i2s_channel_write` — тактируется скоростью DMA, без `vTaskDelay`. Пишет чистый синус: частота `get_param(test_tone_hz)` (440 Гц деф.), амплитуда `master_volume` (0.8), плавный fade-in ~8 мс (антипоп). Параметры читаются с Core 0 lock-free (relaxed atomic) — первый читатель с аудио-ядра.

MCLK не используется (`mclk = I2S_GPIO_UNUSED`) — SCK модуля на GND, внутренний PLL. Пины: **BCK=GPIO5, WS=GPIO6, DIN=GPIO7** (вне strapping/USB-JTAG/PSRAM). I2S-API сверен через Context7 (IDF v6.0), т.к. Espressif MCP в сессии не авторизован; финально подтверждён `make build` на 6.0.2 (`ucsynth.bin` ~228 КБ, чистая сборка).

Железный гайд — `docs/hardware-stage1-pcm5102.md` (пин-в-пин, грабли SCK→GND / XSMT→3.3V / FMT→GND, проверка до кода, дерево диагностики).

**Проверено на железе** (2026-07-15): синус 440 Гц из minijack, чисто, без щелчка (fade-in). Boot: `audio: I2S TX 48000 Гц/16 бит, BCK=5 WS=6 DIN=7`, PSRAM 8 МБ, heap 8523 КБ стабилен (`heap == minheap`). Плата GY-PCM5102A: XSMT=3.3В и FMT=GND — дефолты платы (тыльные площадки), SCK→GND по схеме. **Этап 1.1 закрыт ✅.**

### 2026-07-15 — этап 1 + OLED: инкремент A (scope + дисплей, сборка) 🔨
По просьбе пользователя — отладочный OLED (SSD1306 128×64, I2C 0x3C, модуль GME18864) как временный экран до ST7796 (этап 9). Объединён со звуком, не отдельный этап (долг D-007).

Инкремент A (собрано в облаке, `ucsynth.bin` ~261 КБ):
- **audio scope-буфер:** аудио-задача (Core 0) публикует снимок формы волны (128×int8, до громкости) в двойной буфер (lock-free); `audio_scope_read()` для дисплея.
- **Компонент `display`** (Core 1): свой мини-драйвер SSD1306 (`ssd1306.*`), 1-бит графика + шрифт 5×7 (`gfx.*`), задача рендера ~30 fps. Экран: splash «UCSynth» ~2 с → осциллограф формы + строка статуса (частота) + горизонтальный level bar (master_volume).
- I2C: master-драйвер IDF v6 (`i2c_master.h`, сверено Context7), SDA=8/SCL=9, 400 кГц, 0x3C. Дисплей опционален: при отсутствии ACK система не валится. Долг T-004 (шину вынести в общую).
- Гайд `docs/hardware-oled-ssd1306.md`.

**За пользователем:** подключить OLED (VCC→3.3В, GND, SDA→8, SCL→9), прошить. Ожидаемо: boot `display: OLED SSD1306 128x64 @ 0x3C ...`, на экране splash → осциллограф синуса + level bar. Пусто/тишина — по дереву в гайде.

Дальше (инкременты B+): дизеринг/послесвечение/фейды + param-popup; 1.3 wavetable (форма на осциллографе); 1.2 метрики CPU/underruns.

### 2026-07-15 — этап 1.2: метрики CPU/underruns (сборка) 🔨

На OLED — строка `cpu N.N%  ur M`. `comm` теперь зависит от `audio` (для метрик). Собрано (`ucsynth.bin` ~262 КБ), host-тесты зелёные.

**За пользователем:** прошить; в `serialtest.py`/на экране — `cpu_permille` (для синуса/wavetable — единицы ‰, т.е. доли %), `underruns`=0 (большой запас CPU). Рост cpu / тикающие underruns = сигнал нагрузки (пока не должно).
