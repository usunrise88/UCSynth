### 2026-07-15 — этапы 0.2 + 0.3: модель параметров + бинарный протокол
**0.2 — реестр (`control`):** статический массив по `enum ParamId`, метаданные const (во flash), значения `std::atomic<float>` — lock-free (Core 1 пишет, Core 0 читает без мьютекса). API `set_param`/`get_param`/`param_count`/`param_get_info`. Сид: `master_volume`, `test_tone_hz`.

Кадр `[55 AA][LEN][BODY][CRC16]`, CRC-16/CCITT-FALSE — демукс из ASCII-логов на общем USB-JTAG-канале. Опкоды `GET/SET/LIST/NOTE_ON/NOTE_OFF/STAT`. Транспорт: `usb_serial_jtag` драйвер + задача на **Core 1**, сырое чтение/запись (без stdio/float-printf). Логика (`frame.*`/`protocol.*`) отделена от I/O → host-тестируема. NOTE_ON/OFF пока только ACK+лог (голоса — этап 3).

Проверено: **сборка в облаке** (`ucsynth.bin` ~196 КБ); **host-тест** `tools/run-host-tests.sh` (реестр, кадрирование, CRC, диспетчер; CRC check value `0x29B1` = паритет прошивки с GUI/скриптом). Долг: **D-006** (бинарь), **T-003** (общий канал log+протокол).

### 2026-07-15 — этап 1.1: I2S + PCM5102 ✅ на железе
Аудио-тракт: I2S **standard** TX (Philips, 48 кГц, 16 бит, стерео-слот) на **Core 0**, вывод на PCM5102. Аудио-задача (prio 10, Core 0) генерит блок 64 семпла и блокируется на `i2s_channel_write` — тактируется скоростью DMA, без `vTaskDelay`. Пишет чистый синус: частота `get_param(test_tone_hz)` (440 Гц деф.), амплитуда `master_volume` (0.8), плавный fade-in ~8 мс (антипоп). Параметры читаются с Core 0 lock-free (relaxed atomic) — первый читатель с аудио-ядра.

MCLK не используется (`mclk = I2S_GPIO_UNUSED`) — SCK модуля на GND, внутренний PLL. Пины: **BCK=GPIO5, WS=GPIO6, DIN=GPIO7** (вне strapping/USB-JTAG/PSRAM). I2S-API сверен через Context7 (IDF v6.0), т.к. Espressif MCP в сессии не авторизован; финально подтверждён `make build` на 6.0.2 (`ucsynth.bin` ~228 КБ, чистая сборка).

Железный гайд — `docs/hardware-stage1-pcm5102.md` (пин-в-пин, грабли SCK→GND / XSMT→3.3V / FMT→GND, проверка до кода, дерево диагностики).

**Проверено на железе** (2026-07-15): синус 440 Гц из minijack, чисто, без щелчка (fade-in). Boot: `audio: I2S TX 48000 Гц/16 бит, BCK=5 WS=6 DIN=7`, PSRAM 8 МБ, heap 8523 КБ стабилен (`heap == minheap`). Плата GY-PCM5102A: XSMT=3.3В и FMT=GND — дефолты платы (тыльные площадки), SCK→GND по схеме. **Этап 1.1 закрыт ✅.**

### 2026-07-15 — этап 3.0: нотный путь + band-limited wavetable (сборка) 🔨
Старт этапа 3 (голос целиком). Инкремент 3.0:

- Новый параметр `PARAM_TEST_TONE` (bool) в реестре (реестр 3→4; GUI/serialtest подхватят через LIST). `serialtest.py` играет ноты (три октавы A + высокая пила/меандр — слышно band-limit).

**Фикс (на железе выявлено):** первая версия генерации mip-таблиц считала в `double` → у S3 FPU только float, double шёл софт-эмуляцией (`__muldf3`/…), старт отъедал ~15 с и валил task-WDT. На железе: ноты играют, алиасинга на верхах не слышно (band-limit ок) — подтверждено пользователем.

**За пользователем:** прошить; `serialtest.py` — с загрузки звучит тест-тон 440 (тракт жив), затем `SET test_tone 0` и ноты A3/A4/A5 (высота растёт октавами), высокие ноты пилой/меандром **без «звона»** (band-limit). ### 2026-07-15 — этапы 3.1–3.4 батчем: голос целиком (сборка) 🔨
По просьбе — 3.1–3.4 одним батчем, железная проверка потом скопом. Одиночный осц заменён на
**самодостаточный голос** `voice.*` (моно): 3 осц-слота → микшер (+шум +ring mod осц1×осц2) →
ZDF-фильтр → VCA, две ADSR. Чистый DSP, отделён от I/O (audio.cpp читает control → `VoiceParams`).
- **3.1 ADSR (VCA) + drone** (`env.*`): gate-driven, control-rate (шаг = блок) + пер-семпл лерп VCA
  (анти-зиппер), экспо-сегменты. `latch` (дрон-защёлка: gate = key_down ∨ latch_held, снятие latch
  отпускает), `amp_loop` (цикл A→D→A). Заменил анти-клик рампу 3.0.
- **3.2 фильтр (ZDF SVF) + VCF** (`filter.*`): TPT state-variable (Cytomic/Zavalishin), LP/HP/BP/OFF,
  cutoff+res (k=2−1.98·res, floor запечён), реципрок вынесен из семплового цикла, flush денормалов.
  Вторая ADSR → cutoff экспоненциально (±6 октав, `flt_env_amt`).
- **3.3 три осц + микшер:** на слот форма/детюн(полутоны, дробные=центы)/уровень; шум (xorshift) +
  ring mod. Soft-clip `y/(1+|y|)` в голосе (headroom под сумму голосов 3.5) + hard-clamp на мастере.
- **3.4 lo-fi:** bit-crush (1..16 бит) + принудительный mip0 (без band-limit → алиасинг = фича).
- Реестр **4→31 параметр** (по паттерну `kParams`; GUI/serialtest — через LIST). `display.cpp`:
  снимок popup `prev[16]`→`prev[PARAM_COUNT]` (иначе новые id≥16 не триггерили popup).

Дефолты подобраны так, что **звучит как 3.0**, пока не крутишь (только осц1, фильтр открыт, быстрый

### 2026-07-16 — фикс: кнопки не реагировали на клики (Gio input)
Пользователь: «Кнопки выбора порта, подключения, Panic — все не реагируют на клики». Причина — **экранная
клавиатура перехватывала весь pointer-ввод**: `Keyboard.handleKeys` регистрировала `event.Op(k)` **без clip** →
таг привязывался к корневой (неограниченной) области `-1e6..1e6`. Клавиатура — последний child, её нода в hit-tree
идёт последней → в обратном обходе роутера (`io/input/pointer.go`) она «ловится» первой для **любой** точки и,
будучи non-pass, прыгает к корню, **пропуская все кнопки выше**. Фикс: обернуть `handleKeys` в `clip.Rect` по
прямоугольнику клавиатуры (`app/ui/keyboard.go`) — область ввода ограничена клавиатурой.

Побочно нашёл и исправил вторую ошибку: `handleButtons` был перенесён (в прошлой итерации) **после** layout, но
`material.Button.Layout → Clickable.layout` дренирует и **выбрасывает** все клики в цикле (`widget/button.go`),
поэтому `Clicked()` обязан читаться **до** `Layout`. Вернул `handleButtons` в начало `Layout` (`app/ui/ui.go`).

Регрессия-тест `app/ui/keyboard_test.go` (headless, через `input.Router` — роутинг событий это чистый CPU,
дисплей не нужен): инъекция press+release по кнопке над клавиатурой → клик должен дойти. **Проверено эмпирически:**
без clip тест падает («keyboard input area is shadowing…»), с clip — зелёный. `ui/` добавлен в нативные
`run-app-tests.sh`. `make app-test` зелёный, `.exe` пересобран.
